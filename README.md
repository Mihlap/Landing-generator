# Landing-generator

Генератор одностраничных лендингов с помощью ИИ: описываете бизнес на русском или английском — получаете готовую страницу (HTML), которую можно править в редакторе и при необходимости экспортировать.

## Возможности

- Генерация контента и вёрстки через подключаемые LLM (GigaChat, YandexGPT, OpenAI и др. — см. `.env.example`).
- Два режима сборки лендинга: **полный HTML от модели** или **JSON + шаблоны** на сервере (настраивается переменными окружения).
- Клиент на **React** + **Vite** + **Tailwind**: главная, редактор предпросмотра.
- API на **Node.js** + **Express**: генерация, превью, опционально оплата экспорта через ЮKassa.

## Требования

- **Node.js** 18+ (рекомендуется LTS).
- **npm** (используются workspaces в корне репозитория).

## Установка

```bash
git clone https://github.com/Mihlap/Landing-generator.git
cd Landing-generator
npm install
```

## Переменные окружения

1. Скопируйте пример в корень проекта:

   ```bash
   copy .env.example .env
   ```
   (в macOS/Linux: `cp .env.example .env`)

2. Откройте `.env` и задайте как минимум:

   - **`PORT`** — порт API (по умолчанию `8787`; тот же порт должен совпадать с тем, что читает Vite-прокси).
   - Ключи и настройки хотя бы одного провайдера ИИ (`GIGACHAT_*`, `YANDEX_*`, `OPENAI_API_KEY` и т.д.) — иначе в режиме без ключей будет демо-заглушка.

Подробные комментарии к каждой переменной — внутри [`.env.example`](.env.example).

## Запуск в режиме разработки

Из **корня** репозитория:

```bash
npm run dev
```

Поднимутся:

| Сервис   | URL                    |
|----------|------------------------|
| Фронтенд (Vite) | http://localhost:5173 |
| API (Express)   | http://localhost:8787 (или ваш `PORT`) |

Vite проксирует запросы к API (`/generate`, `/export`, `/preview`, `/health`, `/payments`) на бэкенд.

Отдельно, при необходимости:

```bash
npm run dev -w client
npm run dev -w server
```

## Сборка и продакшен

```bash
npm run build
```

Сначала выполняются тесты (`prebuild`), затем сборка клиента и сервера.

Запуск собранного сервера (раздаёт API; статику фронта при необходимости настройте через nginx или отдельный хостинг):

```bash
npm start
```

Клиент после `build` лежит в `client/dist` — его можно отдавать как статику или использовать `vite preview` в `client`.

## Тесты

```bash
npm test
```

Запускает тесты сервера и клиента по очереди.

## Структура репозитория

```

├── package.json     # workspaces: client, server
└── .env.example     # шаблон переменных окружения
```
